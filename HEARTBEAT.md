# NetOps AI - Heartbeat 自检清单

**目标：** 让项目主动进化，而不是被动等待指令

---

## 每次会话前必读（Agent 启动自检）

```markdown
- [ ] 读取 `SESSION-STATE.md` - 当前任务状态
- [ ] 读取今天/昨天的 memory 文件 - 最近上下文
- [ ] 检查是否有未完成的 TODO
```

---

## 主动检查项（每 24 小时）

### 1. 代码健康度
```bash
# 运行测试
pytest tests/test_nl_router.py -v

# 检查代码质量
ruff check app/nl_router/
ruff format app/nl_router/

# 检查依赖
pip list --outdated
```

### 2. 功能验证
```bash
# 测试自然语言入口
python main_nl.py << EOF
查一下 SW-Core 的配置
给 SW-Core 的 1-4 口配 VLAN 10
quit
EOF
```

### 3. 文档同步
- [ ] README.md 是否反映最新功能
- [ ] SKILL.md 是否需要更新
- [ ] TODO.md 是否需要添加新任务

### 4. 主动改进建议

**如果测试失败：**
→ 创建 issue，分析原因，尝试修复

**如果文档过期：**
→ 自动更新 README/SKILL

**如果发现重复代码：**
→ 提出重构建议

**如果有新意图类型需求：**
→ 记录到 `notes/recurring-patterns.md`

## 主动惊喜项（每周至少 1 个）

**问自己：** "什么功能会让用户说'我没想到这个但太棒了'？"

### 想法池

1. **自动拓扑对比** - 每天自动发现拓扑，对比昨天，高亮变化
2. **配置 drift 检测** - 对比 running-config 和 backup，自动告警
3. **智能诊断报告** - 故障时自动生成 PDF 报告，包含时间线、根因、建议
4. **自然语言批量配置** - "给所有接入交换机的 1-4 口配 VLAN 10"
5. **配置影响分析** - "如果我 shutdown 这个接口，会影响哪些业务？"
